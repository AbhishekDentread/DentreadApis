API Overview

This section provides a top-level view of API and other customizations available to developers.

OAuth API

Dentread uses the OAuth 2.0 authorization framework to issue access tokens on behalf of Dentread Sites.

  • For more on authorization see: OAuth Authorization Docs
  • For authorization-related endpoints see: Authorization API Reference

Dentread is providing 3 public API for developrs.

Authentication API

Dentread provide authentication API for developers and can use easily passing request parameters this is public API endpoints

Files Upload API

Dentread provide file upload API for developers , developers can use this API for upload DICOM files and jpg , png etc. and also pass valid token for autherizaion

Download Files API

Dentread provide file download API for developers , developers can easily download the file useing this API, and also pass valid token for autherizaion

Authorization Using API Key

If you want a simple way to gain access to the API for an individual Dentread Site with the purpose of uploading file and downloading , you can authorize using the Dentread Site's API Key.

File Upload API

Introduction:
  • Dentread provide public File Upload API for developers.
  • Dentread Authentication API is very secure and fast it is based on API key security.
Getting Started:
  • Developers can use easily this API only with pass the request parameters
  • Developers need to send only request parameters Like file
  • Developers can upload zip file but should be Dicomm or stl files inside zip folder
  • Developers can also upload single files like jpeg, jpg, png, html, htm.
API Endpoints:

Endpoint: https://test.dentread.com/uploadfile

  • You need to access this API with http POST method
Request & Response Format
  • You need to pass valid parameters only in Like file.
  • If you pass any wrong and invalid file so API will give you a message Like: file not accepted.
  • All types of validation also there so use this API very carefully.
Api Flow:
  • when you call this API with parameters then only you can access this API
  • And also you have to pass jwt token inside header section.
  • After valid token your file successfully will upload.

// need to pass parameters:

{"file":"file path"}

  • If you passed any wrong parameters and invalid params then you will get a error response
  • Error Response

// need to pass parameters:

{'message': "failed to upload file"}

  • After successfully file upload you will get a API response
  • Success Response

// need to pass parameters:

{'message': 'File uploaded successfully to Azure Server.', 'filename': filename}

You can call this API through this give code.

Example
Copy code

url = "https://cloud.dentread.com/uploadfile"

headers = { "Authorization": "XXXXXXXXXXXXXXXX" }

files = {'file': file}

response = requests.post(url, files=files, headers=headers)

Download API

Introduction:
  • Dentread provide public File Upload API for developers.
  • Dentread authentication API is very secure and fast it is based on API key security.
Getting Started:
  • Developers can use easily this API only with pass the request parameters
  • Developers need to send only request parameters Like:

{"type":1}{"type":2}{"type":3}

and

{"id":1}

  • You need to pass request parameters for download the files
  • If you pass any wrong values then you will get error response
  • If you are downloading zip file and contain Dicomm files then need to pass {"type":2} and {"id":1} in request parameters
  • If you are downloading zip file and contain stl files then need to pass {"type":1} and {"id":1} in request parameters
  • If you are downloading single files like: (png,jpeg,jpg,html,htm) then need to pass {"type":3} and {"id":1} in request parameters
API Endpoints:

Endpoint: https://test.dentread.com/downloanfile

  • You need to access this API with Http GET method
Request & Response Format:
  • You need to pass valid parameters only in Like.
  • If you pass any wrong and invalid file so api will give you a message Like: failed to download file.
  • All types of validation also there so use this API very carefully.
API Flow:
  • When you call this API with parameters then only you can access this API.
  • And also you have to pass jwt token inside header section.
  • After valid token your your file be downloaded.

// need to pass parameters:

{"type":1}{"id":1}

  • If you passed any wrong parameters and invalid params then you will get a error response
  • Error Response

// need to pass parameters:

  • After successfully file downloading you will get a API response

You will get file object
